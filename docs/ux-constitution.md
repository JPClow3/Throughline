# UX Constitution

## Audience

Optimize for student life first. The user is managing classes, assignments, review sessions, exams, and personal study pressure.

## Core Loops

- Capture quickly.
- Break an end goal into small, real tasks (A -> A1..A5) and watch roll-up progress.
- Capture notes and link them to tasks and goals.
- Organize by project/area (optional).
- Move through the board.
- Watch due dates.
- Complete tasks calmly.
- Optionally earn XP / momentum when the game layer is on.
- Export important due work to a calendar.

## Gamification Rules (optional layer)

- Gamification is **off by default**. It appears only when `showGameLayer` is enabled in Settings.
- When on: use XP, level, attributes, badges, and streaks to motivate, surfaced as a quiet momentum strip and a per-card XP chip.
- Do not punish missed days. Streaks celebrate momentum, not shame.
- The calm planner must remain fully usable and uncluttered with the game layer off.
- Completion should feel satisfying but not noisy: short motion, a progress increment, and a subtle 3D pulse only.

## Workflow Priority

1. Fast task capture.
2. Kanban movement.
3. Due-soon awareness.
4. Course context.
5. Calendar export.
6. Notification setup.
7. Visual comfort and 3D progress ambience.

## Chosen V1 Feel

- North star: calm planner.
- Density: spacious and cinematic.
- 3D role: progress layer, not navigation or task management.

## Notification UX

- Be honest about permission state and browser support.
- Never imply push is active when VAPID or service worker setup is missing.
- Local and in-app behavior should still work when push is unavailable.
